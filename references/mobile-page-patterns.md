# Mobile Page Patterns

Use these defaults when the user asks for a mobile prototype that should feel structured and product-ready but does not specify the full page structure.

## Mobile List Page

Default sections:

1. top navigation
2. search, tabs, or filter strip
3. list or card content
4. optional bottom action area

Prompt snippet:

```text
Create a mobile list page with a stable top nav, a light filter area, compact cards or list items, clear status labels, and a practical content rhythm.
```

## Mobile Detail Page

Default sections:

1. top navigation
2. summary card
3. grouped information blocks
4. related content or tabs
5. action area

Prompt snippet:

```text
Create a mobile detail page with a concise summary area, grouped information cards, related sections, and a clear action region.
```

## Mobile Form Page

Default sections:

1. top navigation
2. grouped form sections
3. helper or validation text
4. fixed bottom primary action

Prompt snippet:

```text
Create a mobile form page with grouped fields, compact controls, clear validation, and a fixed bottom action area for submission.
```

## Result Or State Page

Default sections:

1. top navigation
2. status illustration or icon
3. short title and message
4. primary and secondary actions

Prompt snippet:

```text
Create a mobile result page with a calm status presentation, short explanatory text, and clear follow-up actions.
```
