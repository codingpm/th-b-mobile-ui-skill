# Publishing Kit

Use this file as source material when publishing the skill to a skills platform, writing a catalog entry, or preparing example prompts.

## One-Line Description

Generate enterprise mobile UI prototypes with structured page patterns, practical interaction rules, and reusable mobile interface semantics.

## Short Description

Create mobile product prototypes that follow enterprise app patterns such as card lists, grouped forms, fixed bottom actions, status pages, and structured detail views.

## Longer Description

This skill helps Codex generate mobile prototypes, wireframes, Figma prompts, and frontend code that look and behave like a mature enterprise mobile product. The skill encodes common mobile page structures, component-family semantics, and interface rules so the output feels closer to a real app experience and less like a generic consumer mockup.

## Key Selling Points

- Use reusable mobile page patterns for list, detail, form, result, and state pages
- Favor compact app-style layouts over decorative consumer UI
- Prefer structured cards, selectors, grouped forms, and fixed bottom actions
- Include color, button, card, form, and interaction guidance
- Work for design briefs, Figma prompts, HTML, Vue, React, and other prototype outputs

## Best Fit Scenarios

- Generate a mobile list page
- Create a mobile approval or submission form
- Design a mobile detail page with grouped information blocks
- Build a result page or state page
- Draft an enterprise mobile workflow with bottom actions

## Positioning Notes

When presenting this skill publicly:

- describe it as an enterprise mobile UI skill
- emphasize reusable mobile page patterns and interface rules
- highlight cards, grouped forms, selectors, fixed bottom actions, and state pages
- avoid framing the skill as tied to a specific unpublished technical implementation
