# Mobile UI Spec

Use this reference when the task needs a more explicit mobile interface specification.

## Color

- light neutral page background
- white or near-white cards
- dark neutral primary text
- medium neutral secondary text
- one accent color for primary actions
- restrained semantic colors for success, warning, and error

## Buttons

- one clear primary action per main area
- use fixed bottom primary actions for important flows
- keep labels short
- preserve width during loading

## Forms

- use single-column flow
- group fields into cards or sections
- prefer selectors and pickers for structured inputs
- place validation close to the field

## Cards And Lists

- keep hierarchy obvious
- use short supporting text
- use tags or badges for status
- avoid overloading a single card

## Interaction

- bottom sheet or popup for short selection tasks
- full page for complex forms
- segmented tabs for peer categories
- explicit loading, empty, and error states

## Prompt Add-On

```text
Apply structured mobile UI rules: compact spacing, neutral surfaces, one accent color, grouped content cards, clear bottom actions, concise mobile typography, and explicit app-like state feedback.
```
